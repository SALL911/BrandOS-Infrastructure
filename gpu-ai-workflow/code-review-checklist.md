# 代碼審查工作流

## 🔍 代碼審查清單

### 1. 程式碼品質檢查

#### 基礎規範
- [ ] **PEP 8 合規性**: 程式碼符合 Python 風格指南
- [ ] **型別提示**: 所有函數都有完整的 type hints
- [ ] **文檔字串**: 遵循 Google/NumPy 風格的 docstring
- [ ] **命名規範**: 變數、函數、類別命名清晰且一致
- [ ] **匯入順序**: 按照標準庫、第三方函式庫、本地模組排序

#### 錯誤處理
- [ ] **異常捕獲**: 適當的 try-catch 區塊
- [ ] **錯誤日誌**: 詳細的錯誤記錄
- [ ] **資源清理**: finally 區塊或 context manager 使用
- [ ] **自訂異常**: 合理的自訂異常類別

#### 效能考量
- [ ] **記憶體管理**: 及時釋放不需要的資源
- [ ] **演算法複雜度**: 避免不必要的 O(n²) 操作
- [ ] **快取機制**: 適當使用快取減少重複計算
- [ ] **並行處理**: 正確使用 threading/asyncio

### 2. 安全性檢查

#### 輸入驗證
- [ ] **參數檢查**: 函數輸入參數驗證
- [ ] **類型檢查**: 避免類型混淆攻擊
- [ ] **邊界檢查**: 陣列索引、列表長度檢查
- [ ] **SQL 注入**: 使用參數化查詢

#### 資料保護
- [ ] **敏感資訊**: 避免硬編碼密碼、API key
- [ ] **資料加密**: 敏感資料傳輸加密
- [ ] **存取控制**: 適當的權限檢查
- [ ] **日誌安全**: 避免在日誌中洩漏敏感資訊

### 3. 測試覆蓋率

#### 單元測試
- [ ] **函數覆蓋**: 所有公共函數都有測試
- [ ] **邊界測試**: 極值、空值、錯誤情況測試
- [ ] **Mock 使用**: 正確使用 mock 物件
- [ ] **斷言清晰**: 測試失敗時有明確的錯誤訊息

#### 整合測試
- [ ] **API 測試**: 所有 API 端點測試
- [ ] **資料庫測試**: 資料庫操作測試
- [ ] **第三方服務**: 外部服務整合測試
- [ ] **錯誤流程**: 異常情況處理測試

### 4. 架構設計

#### 模組化
- [ ] **單一職責**: 每個模組職責明確
- [ ] **依賴注入**: 避免硬編碼依賴
- [ ] **介面設計**: 清晰的模組介面
- [ ] **擴展性**: 易於擴展的設計

#### 設計模式
- [ ] **工廠模式**: 物件建立統一管理
- [ ] **觀察者模式**: 事件驅動架構
- [ ] **策略模式**: 演算法可替換設計
- [ ] **單例模式**: 全域唯一實例管理

## 📋 審查流程

### Phase 1: 自我審查 (開發者)

#### 準備階段
1. **程式碼整理**: 確保程式碼格式化完成
2. **測試執行**: 所有測試通過
3. **文檔更新**: 相關文檔已更新
4. **日誌檢查**: 無除錯程式碼殘留

#### 檢查清單
```bash
# 程式碼格式檢查
black --check src/
flake8 src/

# 測試覆蓋率檢查
pytest --cov=src --cov-report=term-missing

# 安全性檢查
bandit -r src/

# 型別檢查
mypy src/
```

### Phase 2: 同儕審查 (審查者)

#### 審查重點
1. **功能正確性**: 邏輯實現是否符合需求
2. **程式碼可讀性**: 是否易於理解和維護
3. **效能影響**: 是否有效能瓶頸
4. **安全風險**: 是否存在安全漏洞

#### 審查報告範本
```markdown
## 審查報告

### 總體評價
- [x] 通過
- [ ] 需要修改
- [ ] 拒絕

### 主要問題
1. **問題描述**: 具體問題說明
   - **嚴重程度**: High/Medium/Low
   - **建議修改**: 具體修改建議

### 次要問題
1. **程式碼風格**: 格式、命名等問題
2. **文檔缺失**: 缺少必要的註解或文檔

### 建議
1. **最佳實踐**: 可以改進的地方
2. **學習資源**: 相關學習材料
```

### Phase 3: 修改確認 (開發者 + 審查者)

#### 修改流程
1. **問題修正**: 根據審查意見修改程式碼
2. **回應說明**: 對每個問題的修改說明
3. **重新測試**: 確保修改沒有引入新問題
4. **最終確認**: 審查者確認修改完成

### Git Hooks 設置
```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/psf/black
    rev: 22.3.0
    hooks:
      - id: black
        language_version: python3.8

  - repo: https://github.com/pycqa/flake8
    rev: 4.0.1
    hooks:
      - id: flake8
        args: [--max-line-length=88]

  - repo: https://github.com/pre-commit/mirrors-mypy
    rev: v0.950
    hooks:
      - id: mypy
        additional_dependencies: [types-all]

  - repo: https://github.com/PyCQA/bandit
    rev: 1.7.4
    hooks:
      - id: bandit
        args: [-r, src/]
```
